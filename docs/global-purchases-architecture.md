# /procurements — глобальные закупки

> Last updated: 2026-05-18
>
> Status: первый рабочий слой раздела закупок.
>
> Related issue: #132.

## Назначение

`/procurements` показывает закупки текущей рабочей области. Раздел нужен для ежедневного внесения фактических закупок по материалам, привязки строк к объектам и просмотра закупок за выбранный период.

Главное правило текущего этапа: раздел работает с реальными данными, но не закрывает весь будущий контур закупок. Плановые значения оставлены для следующих этапов. Текущий ручной ввод из справочника материалов заполняет фактические значения.

## Текущий пользовательский сценарий

При открытии `/procurements` список не показывает закупки за все периоды. По умолчанию открывается только текущий день.

Пользователь может:

```txt
открыть сегодняшние закупки
найти закупку по тексту
выбрать другой период через календарь
отфильтровать строки по объекту
добавить несколько материалов подряд из справочника
ввести количество перед добавлением материала
изменить фактическое количество в строке
изменить фактическую цену в строке
выбрать объект прямо в строке
выбрать дату прямо в строке
заменить материал в существующей строке
удалить строку из обычного списка
перейти на следующую или предыдущую страницу списка
```

## Поведение по датам

Даты работают так:

```txt
если дата в адресе не указана → используется сегодня
если выбрана дата или период → список показывает выбранный диапазон
если нажать Сегодня в календаре → фильтр возвращается к текущему дню
новая закупка из окна добавления получает дату сегодня
```

Это поведение задано и на клиенте, и на сервере. Даже прямой запрос списка без `dateFrom/dateTo` вернёт только текущий день.

## Добавление закупок

Добавление идёт только через справочник материалов.

Порядок действий:

```txt
пользователь нажимает Закупка
открывается окно выбора материалов
окно не меняет высоту при поиске и результате
пользователь вводит минимум 2 символа
поиск материалов идёт по быстрому подготовленному поисковому полю справочника
пользователь нажимает Добавить у материала
открывается маленькое окно количества
пользователь вводит количество
после подтверждения строка добавляется в закупки
окно количества закрывается
окно выбора материалов остаётся открытым
пользователь может сразу добавить следующий материал
```

При добавлении из материала заполняются поля:

```txt
title           название материала
unit            единица измерения материала
planQuantity    0
planPrice       цена материала
factQuantity    количество из окна ввода
factPrice       цена материала
purchaseDate    сегодня
projectId       пусто
status          planned
```

Важно: количество из окна добавления — это фактическое количество. Плановое количество не заполняется ручным добавлением. План будет использоваться в следующих этапах, когда закупки будут приходить из плановых источников.

## Замена материала в строке

В меню строки есть действие `Заменить`.

При замене открывается то же окно выбора материалов, но без окна количества. После выбора нового материала окно закрывается.

При замене меняются:

```txt
title
unit
planPrice
```

Сохраняются из старой строки:

```txt
planQuantity
factQuantity
factPrice
projectId
purchaseDate
status
notes
```

## Редактирование строки

В строке отображаются колонки:

```txt
Наименование
Ед. изм
Кол-во План/Факт
Цена План/Факт
Объект, дата, действие
```

Правила редактирования:

```txt
Плановое количество не редактируется
Плановая цена не редактируется
Фактическое количество редактируется через editable badge
Фактическая цена редактируется через editable badge
Объект выбирается из списка объектов
Дата выбирается через календарь
Удаление доступно из меню строки
Замена материала доступна из меню строки
```

Фактические поля сохраняются с короткой паузой. Если пользователь быстро меняет значение, сохраняется последнее состояние, а не каждый промежуточный ввод. Во время сохранения поле не блокируется.

## Группировка и порядок списка

Список отсортирован и визуально сгруппирован по объектам.

Порядок строк:

```txt
объект
дата
наименование
id
```

Строки без объекта попадают в группу `Без объекта`.

## Поиск и фильтры

Фильтры первого этапа:

```txt
поиск по тексту
объект
период дат
```

Статус в данных остаётся, но в пользовательском сценарии текущего этапа не является основным фильтром. Он оставлен для будущих этапов.

## Удаление

Удаление из текущего интерфейса мягкое. Строка не удаляется физически из базы, а получает отметку `archivedAt`. Обычный список такие строки не показывает.

## Быстрый поиск материалов для окна закупок

Окно выбора материалов не использует полный ответ справочника материалов. Для него есть отдельный лёгкий запрос, который возвращает только нужные поля:

```txt
id
title
unit
planPrice
category
```

Поиск работает через подготовленное поле справочника материалов:

```txt
search_fts
```

Для него используется индекс:

```txt
idx_directory_materials_search_fts
```

Такой поиск нужен только для окна закупок и не меняет основной экран справочника материалов.

## Данные закупки

Основная таблица:

```txt
public.global_purchases
```

Основные поля:

```txt
id
workspace_owner_id
title
normalized_title
unit
plan_quantity
plan_price
fact_quantity
fact_price
supplier_id
supplier_name
project_id
project_title
purchase_date
status
notes
search_text
created_by
updated_by
created_at
updated_at
archived_at
deleted_at
```

Поля `supplier_id` и `supplier_name` оставлены под будущую связь с поставщиками. В текущем экране поставщик не выбирается.

`project_title` хранит отображаемую копию названия объекта. Это позволяет показывать название объекта даже если связь будет очищена при удалении объекта.

## Доступ

Чтение доступно пользователю с доступом к текущей рабочей области.

Изменение доступно ролям:

```txt
owner
admin
manager
```

Клиент не передаёт рабочую область как источник истины. Сервер сам определяет текущую рабочую область текущего пользователя.

## Запросы раздела

```txt
GET    /api/global-purchases
POST   /api/global-purchases
GET    /api/global-purchases/material-options
GET    /api/global-purchases/[id]
PATCH  /api/global-purchases/[id]
DELETE /api/global-purchases/[id]
```

Назначение:

```txt
/api/global-purchases                    список и создание закупок
/api/global-purchases/material-options   лёгкий поиск материалов для окна добавления/замены
/api/global-purchases/[id]               чтение, изменение и мягкое удаление одной закупки
```

## Обновление интерфейса

После изменения строки список не перезапрашивается целиком. Обновляется только затронутая строка в текущем списке.

Поведение:

```txt
добавление строки → новая строка вставляется в текущий список, если подходит под фильтры
изменение строки → обновляется только эта строка
удаление строки → строка сразу исчезает из текущего списка
изменение факта → значение показывается сразу, сохранение идёт без блокировки поля
```

## Индексы закупок

Индексы, добавленные для текущего слоя:

```txt
idx_global_purchases_workspace_project_sort
idx_global_purchases_project_id
idx_global_purchases_created_by
idx_global_purchases_updated_by
```

`idx_global_purchases_workspace_project_sort` поддерживает основной порядок списка: объект → дата → название.

Остальные индексы закрывают связи закупок с объектами и пользователями.

После накопления реальных данных нужно отдельно проверить, какие индексы реально используются, а какие можно убрать.

## Дизайн-система

Экран должен собираться из общих компонентов `components/ui` там, где это возможно.

Используются:

```txt
Button
ButtonGroup
Badge
Card
Dialog
DropdownMenu
Popover
Calendar
Input
Skeleton
Empty
FieldError
EditableBadge
```

Неиспользуемые локальные заготовки, оставшиеся от раннего макета, должны быть удалены:

```txt
global-purchases-name.tsx
global-purchases-value.tsx
global-purchases-metric-group.tsx
```

Подтверждение удаления должно быть сделано через общий `Dialog`, а не через `window.confirm`.

## Что не входит в текущий этап

Не реализуется сейчас:

```txt
импорт закупок
экспорт закупок
AI-поиск по закупкам
складской учёт
платежи
логистика
массовые действия
выбор поставщика
сравнение коммерческих предложений
автоматическое создание закупок из смет
ручное создание закупки без выбора материала
редактирование плановых значений вручную
```
