# /procurements — импорт и экспорт закупок

> Last updated: 2026-05-18
>
> Status: первый рабочий слой импорта и экспорта для глобальных закупок.

## Решение по формату экспорта

CSV не подходит как основной формат для передачи закупок в логистику, потому что CSV хранит только значения и не хранит нормальное оформление таблицы.

Основной экспорт закупок делается как настоящий Excel-файл:

```txt
format = xlsx
content type = application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
```

Файл содержит оформленную таблицу:

```txt
заголовок документа
дату выгрузки
количество строк
шапку таблицы
строки закупок
итоговую строку
границы ячеек
числовое выравнивание
закреплённую шапку
фильтр по колонкам
```

## Что попадает в экспорт

Экспорт использует те же активные фильтры, что и экран закупок:

```txt
поиск
объект
период дат
статус, если он появится в адресе
сортировка
```

Значения страницы не используются:

```txt
cursor
limit
```

Сервер собирает строки партиями и ограничивает выгрузку верхним пределом, чтобы не отдавать бесконечный файл.

## Адрес экспорта

```txt
GET /api/global-purchases/export?format=xlsx
```

Кнопка `Экспорт` в верхней панели добавляет текущие фильтры из адреса страницы и скачивает файл.

## Импорт

Первый рабочий слой импорта принимает CSV-файл и показывает предпросмотр перед применением.

Ожидаемые колонки:

```txt
Наименование
Ед. изм.
Кол-во факт
Цена факт
Объект
Дата
Примечание
```

Дополнительно поддерживаются плановые колонки, если они есть:

```txt
Кол-во план
Цена план
```

Если плановые значения не переданы:

```txt
Кол-во план = 0
Цена план = Цена факт
```

## Проверка строк импорта

Перед применением каждая строка проверяется отдельно.

Ошибками считаются:

```txt
нет наименования
нет единицы измерения
некорректное фактическое количество
некорректная фактическая цена
объект указан, но не найден в текущей рабочей области
```

Строки с ошибками показываются в предпросмотре и не применяются.

Корректные строки создаются через существующее создание закупки. Рабочая область и права доступа всё так же проверяются на сервере.

## Ограничения текущего слоя

Текущий импорт не является большим фоновым импортом и не создаёт отдельную задачу импорта.

Не входит в этот слой:

```txt
импорт xlsx
фоновая очередь импорта
обновление существующих закупок по ключу
сопоставление материалов по справочнику
сопоставление поставщиков
```

Эти пункты можно добавлять следующими этапами, если сценарий логистики потребует более строгий обмен файлами.
