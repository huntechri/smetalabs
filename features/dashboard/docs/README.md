# Дашборд (dashboard)

> Статус: production | 2026-05-30

## Назначение

Главная страница приложения после входа. Отображает финансовые показатели воркспейса (KPI-карточки), интерактивный график денежного потока (приход, расход, накопительный баланс) и таблицу активных проектов в работе. Модуль интегрирован с базой данных и API проектов.

---

## Структура модуля

Модуль спроектирован по четырёхслойному архитектурному стандарту:

```txt
features/dashboard/
├── api/
│   └── dashboard-api.ts                  # Реэкспорт методов fetchWorkspaceDashboardStats и fetchProjects
├── model/
│   ├── dashboard-model.ts                # Чистая доменная логика: расчет границ графиков, форматирование дат/меток
│   └── dashboard-model.test.ts           # Модульные тесты для доменной логики
├── application/
│   ├── use-workspace-dashboard-stats.ts  # Хук загрузки KPI и ежедневной статистики транзакций
│   └── use-active-projects.ts            # Хук получения списка проектов в статусе in_progress
└── ui/
    ├── chart-area-client.tsx             # Клиентская ленивая загрузка интерактивного графика (dynamic)
    ├── chart-area-interactive.tsx        # Компонент интерактивного графика Recharts
    ├── data-table.tsx                    # Таблица проектов в работе на базе TanStack Table
    └── section-cards-dashboard.tsx       # Адаптивная сетка KPI карточек воркспейса
```

---

## Взаимодействие слоев и Flow данных

1. Компонент **`page.tsx`** рендерит компоненты верхнего уровня из слоя `ui/`: `SectionCards`, `ChartAreaInteractive` и `DataTable`.
2. Компоненты запрашивают данные через хуки слоя **`application/`** (`useWorkspaceDashboardStats`, `useActiveProjects`).
3. Хуки обращаются к методам слоя **`api/`** (`dashboard-api.ts`), которые выполняют асинхронные HTTP/Supabase запросы к проектам.
4. Полученные сырые данные транзакций и дат форматируются и рассчитываются с помощью чистых функций из слоя **`model/`** (`dashboard-model.ts`) перед рендерингом в JSX.

---

## Описание слоев

### 1. API (`api/dashboard-api.ts`)
Инкапсулирует доступ к данным. В текущей реализации реэкспортирует методы из модуля проектов:
* `fetchWorkspaceDashboardStats` — получение баланса, сметных сумм и истории транзакций воркспейса.
* `fetchProjects` — получение перечня проектов с фильтрацией по статусу.

### 2. Доменная модель (`model/dashboard-model.ts`)
Содержит чистые TypeScript-функции без зависимостей от React и инфраструктуры:
* `formatDateRange(start, end)` — безопасное форматирование диапазона дат проведения проекта (например, `01.05.2026 – 15.05.2026`).
* `prepareChartData(chartData)` — подготовка данных для ComposedChart (инвертирование расходов в отрицательные значения).
* `calculateChartBounds(processedChartData)` — расчет минимальной и максимальной границ по оси Y с учетом 5%-го отступа, а также вычисление точки смещения (`offset`) градиента для заливки баланса.
* `formatYAxisTick(value)` — сокращенное форматирование денежных сумм для шкалы Y (например, `1.5 млн` или `2 тыс`).

### 3. Сценарии / Юзкейсы (`application/`)
* **`useWorkspaceDashboardStats(timeRange)`**:
  * Обертка над TanStack Query `useQuery` с ключом `workspaceStats`.
  * Вызывает `processChartData` для агрегации истории транзакций по дням с фильтрацией по периодам (`7d`, `30d`, `90d`).
* **`useActiveProjects()`**:
  * Загружает проекты со статусом `in_progress` (до 100 записей).

### 4. Интерфейс (`ui/`)
* **`SectionCards`** — 4 карточки: Договор (сумма смет), Оплачено (фактические платежи), Баланс и Отклонение от плана.
* **`ChartAreaInteractive`** — визуализирует приход (столбцы), расход (столбцы вниз) и баланс (линейный график с градиентной заливкой). Поддерживает переключение интервалов.
* **`DataTable`** — отображает список проектов с адресом, сроками и ссылками быстрого перехода.
