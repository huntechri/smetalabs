# Дашборд (dashboard)

> Статус: production (статические/демо-данные) | 2026-05-22

## Назначение

Главная страница приложения после входа. Отображает агрегированные показатели (KPI-карточки), интерактивный график посещаемости и таблицу с данными. Компоненты работают с захардкоженными демо-данными — подключения к реальной БД нет.

## Структура модуля

```
features/dashboard/
  section-cards-dashboard.tsx   — 4 KPI-карточки (выручка, клиенты, аккаунты, рост)
  chart-area-client.tsx         — ленивая загрузка графика (dynamic import + Skeleton)
  chart-area-interactive.tsx    — интерактивный area chart (recharts) с фильтром по датам
  data-table.tsx                — таблица с drag-and-drop, сортировкой, пагинацией, фильтрацией

app/(main)/dashboard/
  page.tsx                      — страница дашборда (Client Component, dynamic='force-dynamic')
  data.json                     — демо-данные для DataTable (29 строк)
  page.tsx.recharts             — бэкап/альтернативная версия страницы
```

## Данные

Подключение к БД для дашборда **отсутствует**. Все данные — статические:

- **`data.json`** (29 строк) — JSON-массив объектов с полями `id`, `header`, `type`, `status`, `target`, `limit`, `reviewer`. Используется компонентом `DataTable`.
- **График** (`chart-area-interactive.tsx`) — встроенный массив `chartData` из 91 элемента (даты с 2024-04-01 по 2024-06-30, поля `desktop` и `mobile`).
- **KPI-карточки** (`section-cards-dashboard.tsx`) — захардкоженные значения: `$1,250.00`, `1,234`, `45,678`, `4.5%`.

## API

API-эндпоинтов для дашборда нет. Страница `app/(main)/dashboard/page.tsx` не делает fetch-запросов — импортирует `data.json` напрямую и передаёт его в `DataTable`.

## Компоненты

### `SectionCards` (`section-cards-dashboard.tsx`)

Четыре карточки в адаптивной сетке (`grid-cols-1`, `@xl/main:grid-cols-2`, `@5xl/main:grid-cols-4`):

| Карточка | Значение | Тренд |
|----------|----------|-------|
| Общая выручка | $1,250.00 | +12.5% ↑ |
| Новые клиенты | 1,234 | -20% ↓ |
| Активные аккаунты | 45,678 | +12.5% ↑ |
| Темп роста | 4.5% | +4.5% ↑ |

Использует `@phosphor-icons/react` (TrendUp, TrendDown), `Card`, `Badge` из shadcn/ui.

### `ChartAreaInteractive` (`chart-area-interactive.tsx`)

Интерактивный area chart на базе **recharts**:

- **Данные:** 91 точка (апрель–июнь 2024), два ряда: `desktop` (ПК) и `mobile` (Мобильные).
- **Фильтр по периоду:** `ToggleGroup` (десктоп) / `Select` (мобильные) — 90d, 30d, 7d.
- **На мобильных:** автоматически переключается на 7d через `useIsMobile()` + `useEffect`.
- **Визуализация:** градиентная заливка (`linearGradient`), тултипы, сетка.
- **Ленивая загрузка:** `chart-area-client.tsx` оборачивает в `dynamic(() => import(...), { ssr: false })` с fallback-скелетоном (`Skeleton className="h-[350px] w-full"`).

### `DataTable` (`data-table.tsx`)

Полнофункциональная таблица на базе **@tanstack/react-table** + **@dnd-kit**:

- **Столбцы:** drag handle, checkbox (выбор строк), Заголовок, Тип раздела, Статус, Цель, Лимит, Проверяющий, Действия.
- **Сортировка:** по всем столбцам (getSortedRowModel).
- **Фильтрация:** по столбцам (getFilteredRowModel, getFacetedRowModel).
- **Пагинация:** 10/20/30/40/50 строк на страницу, навигация.
- **Drag-and-drop:** строки можно перетаскивать для изменения порядка (dnd-kit sortable).
- **Выбор столбцов:** dropdown для скрытия/показа колонок.
- **Вкладки (tabs):** Содержание, Прошлые результаты, Ключевой персонал, Основные документы (последние три — пустые заглушки).
- **Детальный просмотр:** клик по заголовку строки открывает `Drawer` (справа/снизу) с мини-графиком и формой редактирования.
- **Валидация:** `zod` schema для типизации строк.
- **Уведомления:** `sonner` toast при сохранении цели/лимита.
- **Действия:** dropdown-меню (Редактировать, Копировать, Избранное, Удалить).

## Хуки

В `features/dashboard/` специализированных хуков нет. Используются общие хуки проекта:

- `useIsMobile()` из `hooks/use-mobile.ts` — определяет мобильное устройство.

## Зависимости

```
@tanstack/react-table    — табличная логика (сортировка, фильтрация, пагинация)
@dnd-kit/core            — drag-and-drop строк
@dnd-kit/sortable        — сортируемый контекст для dnd
@dnd-kit/utilities       — CSS-трансформации для dnd
recharts                 — графики (AreaChart, XAxis, CartesianGrid, ChartTooltip)
@phosphor-icons/react    — иконки (TrendUp, TrendDown, DotsSixVertical, Columns и др.)
sonner                   — toast-уведомления
zod                      — валидация структуры данных
```

## Tenant boundary

Дашборд не имеет tenant-изоляции — работает только с демо-данными. При подключении к реальной БД потребуется фильтрация по `workspace_id` или `user_id`.

## Текущие ограничения

1. **Демо-данные:** все показатели, графики и таблица используют захардкоженные значения. Подключения к реальной БД нет.
2. **Нет API:** дашборд не делает fetch-запросов. Для production потребуются API-эндпоинты для агрегации KPI.
3. **Нет фильтрации по периоду для карточек:** KPI всегда показывают одни и те же статические числа.
4. **Пустые вкладки DataTable:** «Прошлые результаты», «Ключевой персонал», «Основные документы» — пустые `<div>`-заглушки.
5. **Формы не сохраняют данные:** редактирование цели/лимита/проверяющего в DataTable вызывает toast, но не пишет в БД.
6. **Нет realtime-обновлений:** данные статичны, нет подписки на изменения.
7. **Страница — Client Component:** `"use client"` на всей странице дашборда. `dynamic = "force-dynamic"` на клиентском компоненте не имеет эффекта (применяется только к Server Components).
