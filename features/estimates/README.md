# Модуль Сметы (Estimates Feature)

Данный модуль содержит фронтенд-логику и UI-компоненты для работы со сметами проекта. Он спроектирован по принципу изолированного модуля (feature-based), где вся специфика смет живет в одной папке.

## 📂 Структура директорий

- `api/` — функции для работы с серверным API (через `fetch`). Здесь живет `project-estimate-content-client.ts`.
- `hooks/` — React Query хуки для загрузки и кэширования данных. Главный хук: `useProjectEstimateContent`, который отвечает за получение данных и их оптимистичное обновление.
- `components/` — общие UI-компоненты для смет (например, навигационные вкладки `EstimateNavigationTabs`).
- `estimate-details/` — ядро редактора сметы. Содержит структуру дерева "Разделы -> Работы -> Материалы".
- `estimate-tabs/` — компоненты для других вкладок внутри смет (например, "Финансы", "Документы").

## 🎨 Дизайн и UI (shadcn/ui)

Все компоненты строго следуют правилам `shadcn/ui` и глобальным правилам проекта:
- **Разметка и отступы:** Управление отступами между элементами происходит через `flex flex-col gap-*` (классы `space-y-*` запрещены).
- **Цвета и темы:** Цвета жестко не задаются. Используются семантические системные токены: `bg-background`, `border-border`, `text-muted-foreground` и т.д.
- **Композиция карточек:** Карточки разделов, работ и материалов строятся на базовых компонентах `<Card>`, `<CardHeader>`, `<CardContent>`.
- **Опасные действия:** Удаление элементов реализовано через официальный `<AlertDialog>`.
- **Поля ввода:** Используются составные элементы `<InputGroup>` и `<FieldGroup>` для аккуратной компоновки и валидации.
- **Иконки:** Внутри кнопок иконки позиционируются через атрибут `data-icon`, без жестких размеров вроде `w-4 h-4`.

## 🧩 Ключевые компоненты

1. **`EstimateEditorView`**
   Корневой компонент редактора. Отвечает за загрузку данных, управление локальным состоянием (поиск, фильтрация) и распределение разделов.

2. **`EstimateSectionCard`**
   Отображает один раздел сметы и управляет добавлением/перемещением работ.

3. **`EstimateWorkCard`**
   Отображает конкретную работу внутри раздела, её коэффициенты и список вложенных материалов.

4. **`EstimateMaterialCard`**
   Конечный узел дерева. Отображает наименование материала, его цену, количество, единицу измерения с миниатюрой изображения материала из справочника (если загружено) и меню действий (редактировать/удалить).

## 🚀 Оптимистичные обновления (Optimistic UI)

Изменения данных (создание, удаление, редактирование материалов и работ) выполняются **оптимистично**:
1. При вызове действия пользователь мгновенно видит изменения на экране (обновляется кэш React Query).
2. Фоном запрос отправляется на сервер (`POST /api/projects/[id]/estimate-records/[recordId]/changes`).
3. Если сервер отвечает ошибкой (или обрывается связь, ошибка `failed to fetch`), локальный кэш автоматически откатывается до предыдущего стабильного состояния, гарантируя консистентность данных.
